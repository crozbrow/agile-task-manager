A modelagem do sistema tem como objetivo representar de forma clara e organizada o funcionamento e a estrutura do Agile Task Manager, facilitando o entendimento do projeto antes e durante a implementação. Essa etapa é fundamental na engenharia de software, pois ajuda a reduzir erros, alinhar requisitos e servir como base para o desenvolvimento e manutenção do sistema.

Foram utilizados diagramas UML, que permitem visualizar tanto o comportamento do sistema quanto sua estrutura interna.

Diagrama de Casos de Uso

O diagrama de casos de uso representa as interações entre o usuário e o sistema, evidenciando as principais funcionalidades disponíveis. Nele, é possível identificar as ações que o usuário pode executar, como criar, listar, atualizar, alterar o status e excluir tarefas. Esse diagrama ajuda a compreender o escopo funcional do sistema de forma objetiva.

Diagrama de Classes

O diagrama de classes descreve a estrutura estática do sistema, destacando suas principais classes, atributos e responsabilidades. Neste projeto, a classe Task representa a entidade central, responsável pelo gerenciamento das tarefas, incluindo informações como identificador, título, descrição e status. Esse diagrama reflete diretamente a implementação realizada na aplicação.

Os diagramas UML utilizados estão disponíveis na pasta /docs do repositório, servindo como apoio à documentação e ao entendimento do sistema.
